# Data Model: F1 Week 8 — Hurdle Model Bayesian Optimisation

**Feature**: `019-f1-week8-hurdle`  
**Date**: 2026-03-01

## Entities

### 1. Observation Data

The core dataset for F1 Week 8.

| Field | Type | Shape | Description |
|-------|------|-------|-------------|
| X | numpy.ndarray (float64) | (18, 2) | Input locations in [0.0, 1.0]² |
| y | numpy.ndarray (float64) | (18,) | Observed output values (radiation intensity) |

**Source**: `./data/f1/updated_inputs - Week 8.npy`, `./data/f1/updated_outputs - Week 8.npy`

**Validation Rules**:
- All input values in [0.0, 1.0]
- No NaN values in inputs or outputs
- Exactly 18 rows (10 initial + 8 weekly submissions)

---

### 2. Derived Binary Labels

Binary classification for the hurdle model's Stage 1.

| Field | Type | Shape | Derivation |
|-------|------|-------|------------|
| y_binary | numpy.ndarray (bool) | (18,) | `y > 0` — True where output is strictly positive |
| n_positive | int | scalar | `y_binary.sum()` — count of positive observations |
| X_pos | numpy.ndarray (float64) | (n_positive, 2) | Input subset where y > 0 |
| y_pos | numpy.ndarray (float64) | (n_positive,) | Output subset where y > 0 |
| y_pos_log | numpy.ndarray (float64) | (n_positive,) | `log1p(y_pos)` — transformed targets for Stage 2 |
| FALLBACK_MODE | bool | scalar | `True` when `n_positive < MIN_POSITIVE (3)` |

---

### 3. Hyperparameter Constants

Named constants controlling the model and acquisition behaviour.

| Constant | Type | Value | Role |
|----------|------|-------|------|
| C_STAGE1 | float | 1.0 | L2 regularisation for logistic regression |
| N_ESTIMATORS | int | 100 | Number of random forest trees |
| MAX_DEPTH | int | 3 | Maximum tree depth (prevents overfitting) |
| KAPPA | float | 3.0 | UCB exploration weight (high = exploration-focused) |
| PENALTY_RADIUS | float | 0.15 | Local penalization Gaussian radius |
| N_CANDIDATES | int | 20000 | Random candidate set size for acquisition maximisation |
| GRID_RES | int | 50 | Visualisation grid resolution per axis |
| MIN_POSITIVE | int | 3 | Minimum positive samples for Stage 2 training |
| STEEPNESS | float | 0.1 | Interior penalty boundary suppression exponent |
| FLOOR | float | 0.01 | Interior penalty minimum value at boundary |

---

### 4. Surrogate Model Outputs

Predictions generated by the fitted hurdle model over the candidate set.

| Field | Type | Shape | Description |
|-------|------|-------|-------------|
| p_cand | numpy.ndarray (float64) | (N_CANDIDATES,) | P(y > 0) from calibrated classifier |
| mu_log_cand | numpy.ndarray (float64) | (N_CANDIDATES,) | Mean log1p(y) prediction from RF |
| sigma_rf_cand | numpy.ndarray (float64) | (N_CANDIDATES,) | Std of per-tree predictions (uncertainty) |
| mu_cand | numpy.ndarray (float64) | (N_CANDIDATES,) | `expm1(mu_log_cand)` — back-transformed mean |

---

### 5. Acquisition Scores

Composite acquisition values combining UCB, local penalization, and interior penalty.

| Field | Type | Shape | Description |
|-------|------|-------|-------------|
| acq_raw | numpy.ndarray (float64) | (N_CANDIDATES,) | `p * mu + KAPPA * p * sigma_rf` |
| penalty | numpy.ndarray (float64) | (N_CANDIDATES,) | Local penalization factor ∈ [0, 1] |
| interior_weight | numpy.ndarray (float64) | (N_CANDIDATES,) | Interior penalty factor ∈ [FLOOR, 1.0] |
| acq_final | numpy.ndarray (float64) | (N_CANDIDATES,) | `acq_raw * penalty * interior_weight` |
| next_x | numpy.ndarray (float64) | (2,) | Proposed next sample point |

---

### 6. Submission Query

The formatted output for weekly submission.

| Field | Type | Format | Constraints |
|-------|------|--------|-------------|
| formatted_query | string | `"0.xxxxxx-0.xxxxxx"` | 6 decimal places, values in [0.0, 0.999999] |

---

## State Transitions

```
Load Data → Validate → Derive Binary Labels
                            ↓
                    n_positive ≥ 3?
                   /              \
                 YES               NO
                  ↓                 ↓
           Fit Stage 1        Fit Stage 1
           Fit Stage 2        FALLBACK_MODE
                  ↓                 ↓
           Weighted UCB      Pure Exploration
                  ↓                 ↓
           Apply Penalties    Apply Penalties
                  ↓                 ↓
           Select Candidate   Select Candidate
                  ↓                 ↓
           Validate Point     Validate Point
                  ↓                 ↓
           Visualise          Visualise
                  ↓                 ↓
           Format Query       Format Query
```

## Relationships

- **Observation Data** → feeds → **Derived Binary Labels** (via `y > 0` threshold)
- **Derived Binary Labels** → determines → **FALLBACK_MODE** (controls Stage 2 availability)
- **Hyperparameter Constants** → configure → **Surrogate Model** and **Acquisition Scores**
- **Surrogate Model Outputs** → feed → **Acquisition Scores** (via weighted UCB formula)
- **Acquisition Scores** → produce → **next_x** → formatted as → **Submission Query**
