# Datasheets

This document provides datasheets for each of the eight black-box optimisation functions (F1–F8) in the capstone project. Datasheets are structured transparency documents, inspired by Gebru et al. (2021), that describe the motivation, composition, collection process, preprocessing, and distribution of a dataset. They support data provenance, informed reuse, and responsible interpretation by readers who may not have direct access to the raw data files or notebooks.

This project tackled eight black-box optimisation problems as part of a certificate course in AI and ML. For each function, an initial dataset was provided by the challenge organisers, and additional data points were acquired through surrogate-model-guided optimisation over twelve weekly submission rounds (Weeks 3–13). The datasheets below document the complete dataset for each function as it stood at the end of Week 13.

---

## F1 — Radiation Source Detection

### Motivation

- **Purpose**: This dataset was created for a black-box optimisation challenge as part of a capstone AI and ML course. The goal is to find the input configuration that maximises the output of a synthetic radiation source detection function.
- **Domain**: Radiation source detection — the function simulates the problem of locating a radiation source in a 2D parameter space, where non-zero output values indicate proximity to the source.
- **Creators**: The initial 10 observations were provided by the challenge organisers. The subsequent 13 data points were generated by the student's surrogate-model-guided optimisation pipeline, with outputs evaluated by the challenge oracle (a black-box function evaluator with no access to function internals).
- **Funding/Context**: Capstone project for a certificate course in AI and ML.

### Composition

- **Input dimensionality**: 2 features
- **Input bounds**: [0, 0.999999] per dimension (normalised)
- **Instances**:
  - Initial: 10 observations
  - Final (Week 13): 23 observations
  - Growth: 13 points added across 12 weekly submissions (Weeks 3–13)
- **Output characteristics**:
  - Sign: Zero-inflated — the vast majority of observations return zero or near-zero
  - Range: [−3.606 × 10⁻³, 7.71 × 10⁻¹⁶]
  - Special properties: Zero-inflated distribution; the radiation source has not been located in the sampled region
- **Missing data**: None — all queries received valid responses
- **Confidentiality**: No personally identifiable information

### Collection Process

- **Initial data**: 10 pre-generated observations provided by the challenge organisers
- **Acquisition strategy**: Weeks 3–9: Hurdle model (logistic classifier + random forest regressor) with weighted UCB. Weeks 10–13: SFGP Matérn-2.5 with qLogNEI. Candidate input(s) submitted → oracle returned true function value(s)
- **Rounds**: 13 total (initial provision + Weeks 3–13)
- **Points per round**: Week 3 submitted 3 candidates; Weeks 4–13 submitted 1 candidate per round
- **Time frame**: Approximately 12 weeks of weekly submissions
- **Oracle**: Black-box function evaluator provided by course organisers (no access to function internals)

### Preprocessing and Uses

- **Input preprocessing**: None — inputs used directly in [0, 0.999999] normalised form
- **Output preprocessing**: Weeks 3–9: log1p(y) transform for positive outputs (hurdle model). Weeks 10–13: BoTorch Standardize transform (GP).
- **Intended use**: Training surrogate models for Bayesian optimisation to locate the radiation source
- **Inappropriate uses**:
  - Direct use as ground truth for real radiation source detection (data represents a synthetic challenge function)
  - Extrapolation beyond the [0, 0.999999] input domain

### Distribution and Maintenance

- **Storage**: `data/f1/` directory in the project repository
- **Format**: NumPy `.npy` files (binary array format)
- **Files**:
  - `initial_inputs.npy` — initial input observations (10, 2)
  - `initial_outputs.npy` — initial output observations (10, 1)
  - `updated_inputs - Week 3.npy` through `updated_inputs - Week 13.npy` — cumulative inputs through each week
  - `updated_outputs - Week 3.npy` through `updated_outputs - Week 13.npy` — cumulative outputs through each week
- **Status**: Static — data collection is complete, no further updates planned
- **Licence/Terms**: Academic use within capstone course
