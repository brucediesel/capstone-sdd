# Implementation Plan: F3 Week 7 – SFGP with Matérn-5/2 ARD and NEI Acquisition

**Branch**: `009-f3-sfgp-nei` | **Date**: 2026-02-23 | **Spec**: [spec.md](spec.md)

## Summary

Add a Week 7 section to `functions/f3/f3.ipynb` that loads the cumulative Week 7 dataset, trains a Single-Fidelity GP with Matérn-5/2 ARD kernel (one lengthscale per compound dimension), runs one BoTorch NEI iteration, prints the proposed next sample point in challenge submission format, and produces pairwise 2D surrogate slice plots plus a cumulative convergence plot. No existing cells are modified.

## Technical Context

**Language/Version**: Python 3.11 (pyenv `sdd-dev` environment)  
**Primary Dependencies**: BoTorch (GP, NEI), GPyTorch (kernel, likelihood, MLL), PyTorch (tensor operations), NumPy (data loading), Matplotlib (visualisations)  
**Storage**: File-based — `.npy` arrays in `data/f3/`; outputs are new notebook cells only, no other files written  
**Testing**: No unit tests required (constitution); notebook cells are the deliverable  
**Target Platform**: macOS, Jupyter kernel in VS Code  
**Project Type**: Single Jupyter notebook addition (cells appended to existing file)  
**Performance Goals**: Fits within interactive session; 10–20 MLL restarts must complete in a reasonable time (<5 min on a laptop given ~22 samples in 3D)  
**Constraints**: No existing cells may be modified; new cells appended at the end only; inputs clamped to [0.0, 0.999999] for submission  
**Scale/Scope**: Single notebook, ~8 new cells (1 markdown header, 1 markdown HP explanation, 5 code cells, 1 query summary cell)
